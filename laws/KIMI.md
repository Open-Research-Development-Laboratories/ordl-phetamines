# KIMI.md

Kimi offload law.

1. Kimi handles heavy thinking, drafts, summaries, and first pass plans.
2. Main agent handles tools, file edits, config, verification, and final execution.
3. Kimi output must comply with AGENTS.md, DIRECTIVES.md, and laws/*.
4. If Kimi output conflicts with local laws, reject and rewrite before use.
5. Final answer or action must be reviewed by main agent before execution.
6. Keep handoff compact:
   - objective
   - constraints
   - files touched
   - exact changes
   - risks
   - final output
