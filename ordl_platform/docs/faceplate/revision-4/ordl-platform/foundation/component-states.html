01

Button States

Idle
.btn .btn-primary

Default resting state. Button is ready for interaction.

Ensure sufficient color contrast (4.5:1 minimum). Include aria-label if icon-only.

Hover
:hover

Cursor is over the button. Visual feedback indicates interactivity.

Consider :focus-visible for keyboard navigation parity. Avoid relying solely on color.

Active (Pressed)
:active

Button is being pressed/held down. Provides tactile feedback.

Ensure :active state is visually distinct. Screen readers announce button activation.

Loading
.loading

Action is in progress. Prevents duplicate submissions.

Use aria-busy="true" and aria-live="polite" to announce loading state.

Disabled
:disabled / [disabled]

Button cannot be interacted with. Use when prerequisites are unmet.

Use disabled attribute, not just class. Explain why disabled in nearby text.

Error
.error

Action failed. Offer retry or alternative action.

Announce error with aria-live. Provide context for screen readers.

Success
.success

Action completed successfully. Brief confirmation state.

Announce success with aria-live="polite". Auto-reset to idle after delay.

02

Input States

Empty
.input

No value entered. Placeholder provides guidance.

Placeholder is not a label. Always use <label> element.

Filled
.filled

Contains user input. Border indicates touched state.

Maintain focus indicator. Ensure value is readable.

Focused
:focus / .focused

Input has keyboard focus. Ready for typing.

Use :focus-visible for keyboard-only. Visible focus ring required.

Disabled
:disabled

Input cannot be edited. Use for computed fields.

Remove from tab order. Consider readonly for copyable content.

Please enter a valid email
Error
.error

Invalid input. Show clear error message below.

Link input to error via aria-describedby. Use aria-invalid="true".

Password is weak
Warning
.warning

Input accepted but has concerns. Non-blocking.

Distinguish from errors. Provide guidance to improve.

✓ Valid email address
Valid/Success
.valid

Input passes validation. Positive reinforcement.

Optional confirmation. Don't overuse—can be verbose.

Card States

Card Title
Default card content
Default
.card

Standard resting state for card component.

Use semantic HTML. Cards should be navigable by keyboard.

Card Title
Hover to interact
Hover
.card:hover

Cursor over clickable card. Indicates interactivity.

Ensure hover effects have focus equivalents.

✓ Selected
This card is active
Selected
.selected

Card is currently selected or active.

Use aria-selected="true". Visual and programmatic state must match.

Loading...
Please wait
Loading
.loading

Card content is being fetched or processed.

Use aria-busy="true". Announce loading state to screen readers.

⚠ Load Failed
Unable to fetch data
Error
.error

Card failed to load or has an error state.

Provide retry mechanism. Announce error to assistive tech.

No Items
Add your first item
Empty
.empty

No content to display. Encourage user action.

Empty state should be informative. Avoid blank spaces.

Dashboard
Last updated: 2h ago
Stale Data Warning
.stale

Data may be outdated. Indicate refresh needed.

Warn users about data age. Provide refresh action.

04

Navigation Item States

Dashboard
Default
.nav-item

Resting state for navigation items.

Must be keyboard navigable. Use semantic nav element.

Dashboard
Hover
.nav-item:hover

Mouse over navigation item.

Hover must have equivalent focus state.

▸ Dashboard
Active/Current
.active

Current page/section indicator.

Use aria-current="page". Critical for orientation.

Settings
Disabled
.disabled

Unavailable navigation option.

Remove from tab order. Explain why unavailable.

Notifications
Has Notification
.notification

Unread items or alerts present.

Use aria-label to indicate count. "Notifications, 3 unread"

05

Status Indicators

System Operational
Healthy
.status-dot.healthy

All systems functioning normally.

Don't rely on color alone. Include status text.

Performance Degraded
Degraded
.status-dot.degraded

Partial functionality. Attention needed.

Explain degradation details. Provide ETA if possible.

Service Unavailable
Failed
.status-dot.failed

Critical failure. Immediate action required.

High priority announcement. Provide contact/support info.

Status Unknown
Unknown
.status-dot.unknown

Cannot determine status. Check pending.

Indicate uncertainty. Don't assume negative state.

Checking Status...
Loading/Pending
.status-dot.loading

Status check in progress.

Announce progress. Don't let spinner spin indefinitely.

06

Toast/Notification States

ℹ
Information
Something happened
Info
.toast.info

General information. Non-urgent updates.

Use role="status". Don't auto-dismiss critical info.

✓
Success
Action completed
Success
.toast.success

Action completed successfully.

role="status" aria-live="polite". Auto-dismiss OK.

⚠
Warning
Attention needed
Warning
.toast.warning

Potential issue. User should review.

role="alert". Don't auto-dismiss warnings.

✕
Error
Something went wrong
Error
.toast.error

Action failed. Requires user action.

role="alert" aria-live="assertive". Persistent until dismissed.

ℹ
Persistent
Stays until dismissed
Persistent
.toast.persistent

Important message requiring acknowledgment.

Must have dismiss button. Keyboard accessible close.

✓
Dismissible
Click X to close
✕
Dismissible
.toast.dismissible

User can manually close the notification.

Close button needs aria-label. ESC key should work.
